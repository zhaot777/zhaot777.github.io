Database Performance-Tuning Review

Posted on 2016-06-19   |   In Database   |  

Performance-Tuning concepts

  1. Set of activities and procedures that reduce response time of database system- Fine -tuning the performance of a system requires that all factors must operate at optimum level with minimal bottlenecks
  • All data in a database are stored in data files, and logical grouped similar characteristics data in file groups or table spaces
  • Data cache or buffer cache: stores most recently accessed data blocks in RAM
  • SQL cache or procedure cache: Stores most recently executed SQL statements or PL/SQL procedures
  • Majority of performance-tuning activities focus on minimizing I/O operations between RAM and permanent storage
  • Automatic query optimization & Manual query optimization
  • Static query optimization & Dynamic query optimization
  • Statistically based query optimization algorithm & Rule-based query optimization algorithm

DBMS processes SQL queries

  1. Query Processing:
    • Parsing: Query Optimizer chooses the most efficient access plan
    • Execution: Reuse the exists cache plan or execute the SQL using the chosen execution plan
    • Fetching: DBMS fetches the data and sends the result set back to the client

      Indexes

  2. Help speed up data access
  • Data sparsity: number of different values a cloumn could have. low sparsity make index might be useless
  • Data structures used to implement indexed:
    • Hash indexed
    • B-tree indexed
    • Bitmap indexed
  • Index Selectivity: Measure of the likelihood that an index will be used in query processing
    • Index cannot always be used to improve performance
    • Function-based index

      Query Optimizer

  1. Optimizer Choices
    • Rule-based optimizer
    • Cost-based optimizer
  • Optimizer hints: special instructions for the optimizer, embedded in the SQL command text

Formulate Queries and tune the DBMS for optimal performance

  • Using RAID(Redundant Array of Independent Disks)
