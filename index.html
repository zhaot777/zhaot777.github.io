What is Big Data

  1. Big Data is data that is ‘too big’ to be stored in a single machine, and/or processed by a single machine.
  • Feature: Variety, Velocity, Volume
    • Encompasses large amount of information
    • Consists of a variety of data types and formats
    • Generated by disparate sources
    • Retained for long periods
    • Utilized by new and innovative applications
  • Sources of Big data: web browsing, purchasing, social media, docs, genomic data, GPS, sensors …
  • Datafication: turning many aspects of our life into computerised forms of new meaning and forms of value.
  • IoT: Internet of Things
  • Issues:
    • Security/Privacy
    • Rigor
    • Timing
    • Comprehensiveness
    • Standards
    • Sampling
    • Utilization
    • Too much data
      • Benefit of Big Data
      • Combine multiple sources of data
      • Exploit unstructured data
      • Provide insights
      • Quicker action

What is Data Science

  1. Math, Algorithms, Coding, Subject matter
  • Concentration areas:
    • Spatial analysis
    • text mining
    • Speech
    • Social media
    • Big Data
    • Medical
    • Bioinformatics
  • Process steps:
    • Question
      • data acquisition
    • Data
      • data processing, data management, data storage, data retrieval
    • Methods
      • data mining, machine learning, regression
    • Inference
      • computational statistics
    • Presentation
      • visualization, data graphic design
    • Answer
  • Tools
    • Data mining tool – Weka
    • Scratch analyses – homegrown and open-source libraries in R, Python, Java etc.
    • Visualizing – Qlikview, Tableau, periscope.io

title: NoSQL Review
tags: Database, NoSQL, Curriculum Review
category: Database

description: Basic concept of NoSQL and 4 types of NoSQL Databases

What is NoSQL

  1. Need for NoSQL
    • Lots of new and new types of data
    • Hard to use a relational model
    • Hard to scale up to fit more data, more users
    • Hard to keep up performance
  • Feature: flexible, schema-less, efficient & fast, available, scalable
  • JSON: a set of key-value pairs
  • ‘BASE’ property
    • Best effort
    • Approximate answer
    • Simpler
    • Easier evolution

NoSQL DBs

  1. Types: Key-value store, Column-family store, Document store, Graph store
  • Standard way to compare NoSQL DBs: Architecture, administration, deployment, development, performance & scalability
  • Polyglot persistence: use different storage technologies to store different parts of a single application and tied together by the application, using DB APIs or web services APIs.
  • NoSQL benefit
    • Improve programmer productivity, better matches needs
    • Improve data access performance, large volumes
  • Node & Cluster: Node is a single NoSQL DB instance, Cluster is a collection of nodes which holds an entire DB.
  • Key-value DBs
    • While DB is a dictionary
    • Records don’t have to contain identical fields
    • Key is comparable to a PK. Value is an aggregate of all the dependent columns.
    • Querying occurs only on keys, and entire value is returned.
    • Memcached: distributed memory object caching system, simple
    • Redis: offers richer types such as list, set and dictionary for values, and AlSO offer a way to use hashing for storing keys
    • Anazon’s Dynamo
  • Column family DBs
    • BigTable, HBase, Cassandra, Amazon SimpleDB, Hypertable.
    • Column family, Column, Super column family
  • Document DBs
    • A more sophisticated version of a key-value store, whre the value is a collection of documents, can be JSON, XML, etc.
  • Graph DBs
    • Use nodes, relations, properties(k-v pairs) on vertices and edges to store data.
    • index free
  • Triple store(Resource Description Framework)
    • subject, predicate, object
    • class, attribute, value
    • Architecture mode: in-memory, native store, non-native store(use third-party service)

Performance-Tuning concepts

  1. Set of activities and procedures that reduce response time of database system- Fine -tuning the performance of a system requires that all factors must operate at optimum level with minimal bottlenecks
  • All data in a database are stored in data files, and logical grouped similar characteristics data in file groups or table spaces
  • Data cache or buffer cache: stores most recently accessed data blocks in RAM
  • SQL cache or procedure cache: Stores most recently executed SQL statements or PL/SQL procedures
  • Majority of performance-tuning activities focus on minimizing I/O operations between RAM and permanent storage
  • Automatic query optimization & Manual query optimization
  • Static query optimization & Dynamic query optimization
  • Statistically based query optimization algorithm & Rule-based query optimization algorithm

DBMS processes SQL queries

  1. Query Processing:
    • Parsing: Query Optimizer chooses the most efficient access plan
    • Execution: Reuse the exists cache plan or execute the SQL using the chosen execution plan
    • Fetching: DBMS fetches the data and sends the result set back to the client

      Indexes

  2. Help speed up data access
  • Data sparsity: number of different values a cloumn could have. low sparsity make index might be useless
  • Data structures used to implement indexed:
    • Hash indexed
    • B-tree indexed
    • Bitmap indexed
  • Index Selectivity: Measure of the likelihood that an index will be used in query processing
    • Index cannot always be used to improve performance
    • Function-based index

      Query Optimizer

  1. Optimizer Choices
    • Rule-based optimizer
    • Cost-based optimizer
  • Optimizer hints: special instructions for the optimizer, embedded in the SQL command text

Formulate Queries and tune the DBMS for optimal performance

  1. Find a good way to construct a SQL query
  • Using RAID(Redundant Array of Independent Disks)

title: Database Transaction Review
tags: Database, Curriculum Review
category: Database

description: Transactions, and different Concurrency control approach.

Database Transactions

  1. Logical unit of work that must be entirely completed or aborted
  • Consistent database state: all data integrity constraints are satisfied
  • Formed by two or more database requests, and it should equivalent of a single SQL statement in an application program or transaction
  • ACID
    • Atomicity
    • Consistency
    • Isolation
    • Durability
  • Transaction Management with SQL
    • COMMIT
    • ROLLBACK
  • Transaction Log
    • Keep track of all transactions that update the database
    • For recovery requirement triggered by a ROLLBACK statement
    • A program’s abnormal termination
    • A system failure

Concurrency control

  1. Ensure serializability of transactions in a multiuser database environment
  • Problems
    • Lost update: update same data
    • Uncommitted data: one accessed uncommitted data, the other one rolled back
    • Inconsistent retrievals: transaction accesses on some data other is working on
  • Serializable schedule: Interleaved execution of transactions yields the same results as the serial execution of the transactions
  • Locking methods
    • Type
      • Binary Lock
      • Exclusive Lock(write)
      • Shared Lock(read)
    • Two-Phase Locking(2PL)
      • Defines how transactions acquire and relinquish locks, guarantees seralizability but does not prevent deadlocks
      • Growing phase - Transaction acquires all required locks without unlocking any data
      • Shrinking phase - Transaction releases all locks and cannot obtain any new lock
      • Rules
        • Two transactions cannot have conflicting locks
        • Cannot unlock before lock
        • No data are affected until all locks are obtained
  • Stamping methods
    • Assigns global, unique time stamp to each transaction
    • Properties: Uniqueness & Monotonicity
    • Disadvantages: increase memory, processing, resources, more fields
  • Optimistic methods
    • Transaction is validate to ensure that the changes made will not affect the integrity and consistency of the database
    • Use data resources without acquiring locks on those resources

title: DB Normalization Review
tags: Database, Curriculum Review
category: Database

description: Basic concept of Database Normalization, normal forms and how to implement each forms.

What is normalization and what role it plays in the DB design process

  1. Evaluating and correcting table structures to minimize data redundancies
  • Reduces data anomalies
  • Higher normal forms are better than lower normals forms and result in cleaner designs
  • Properly designed 3NF structures meet the requirement of 4NF
  • Normalization is a design step: Used to design a new DB structure or to improve the existing data structure
  • Normalization Goal
    • Each table represents a single subject
    • No data item will be unnecessarily stored in more than one table
    • All non-prime attribute in a table are dependent on the primary key
    • Each table is void of insertion, update and deletion anomalies

Normal Forms

  1. Characteristic
    • 1NF: Table format, no repeating groups, PK identified
    • 2NF: 1NF and no partial dependencies
    • 3NF: 2NF and no transitive dependencies
    • 4NF: 3NF and no independent multivalued dependencies
  • Process
    • Work on one relation at a time
    • Identifying the dependencies of a relation
    • Progressively breaking the relation into new set of relations
  • Dependencies
    • Functional Dependence: Each value of attribute A determines one and only one value of attribute B, then we say B is functionally dependent on A
    • Partial Dependency: functional dependence in which the determinant is only part of the PK
    • Transitive Dependency: An attribute functionally depends on another non-key attribute
  • Dependency diagram
    • Mark PK
    • Indicate full dependencies on the top and partial/transitive dependencies on the bottom
  • BCNF
    • Candidate key: same as primary key but not chosen to be the primary key
    • If the table contains more than one candidate key, it violated the BCNF, which is a special case of 3NF
    • Solution: try to change the primary key and separate one relation to two

Denormalization

  • Produces a lower normal form and increased performance and greater data redundancy
